# Lavi: A Language-Native Self-Growing AI Architecture
## Draft — 2026-06-02

## Notation

本論文では 以下の 呼称を 用いる。

- **Lavi**: 本研究が 提案する 自律成長 AI システム。
- **Loa**: 実験環境において 外部評価者・対話相手として 機能する Claude Code CLI エージェント。
- **Gles**: Lavi の 設計者・著者であり、Loa とともに 初期評価者の 役割を 担う ユーザー。

## Abstract

本論文は、Transformer が抱える6つの構造的欠陥を解消することを目指した、言語ネイティブな自己成長型 AI アーキテクチャ「Lavi」の設計提案です。実証実験は未実施であり、本稿はあくまで設計提案論文として位置づけられます。Lavi は4層構造から成ります。L1 は Mamba2 SSM による O(n) の系列処理、L2 は LaEMe (Language-based Evaluative Metadata) による言語ベースの評価的記憶、L3 は Active Inference / DEM による自由エネルギー最小化と予測誤差学習、L4 は Darwin Gödel Machine によるアーキテクチャの自己改善です。LaEMe はあらゆる経験や知覚に自然言語のメタデータを付与する汎用フレームワークであり、MARS の原理・手続き構造を包含します。各欠陥への達成度を、次のように誠実に報告します。すなわち、O(n²) 計算量の問題と記憶構造の欠如は解消、準固定重みと自己説明性の欠如は大幅に緩和、ハルシネーションは緩和、事前学習データ依存は部分的に対処 (獲得後の継続学習として再定義) です。

## Section 1 : Introduction

### トランスフォーマーの 欠陥 (重要度順)

**欠陥 1 : O(n²) 処理複雑度**

標準的な self-attention は O(n²) であり、
長系列処理の 根本的な ボトルネックになる。
Sparse Attention 等の 変種で 緩和できるが、
設計原理の 変更を 伴う。

**欠陥 2 : 記憶構造の 欠如**

継続学習・生涯学習の 実装は 可能だが、
EWC・リプレイバッファ等の 追加機構が 必須となり、
アーキテクチャに 記憶が 設計されていないことによる
構造的 非効率が 残る。

**欠陥 3 : 重みの 準固定性**

重みの 持続的 更新は ファインチューニングを 要し、
コストが 高い。
インコンテキスト学習は セッション内の 一時的 適応に 過ぎず、
永続しない。

**欠陥 4 : 事前学習データへの 構造的 依存**

大量の 事前学習データへの 依存度が 構造的に 高く、
経験だけから 自律的に 言語を 獲得する 設計に なっていない。

**欠陥 5 : ハルシネーション**

生成機構 自体に 意味的 正確性の 保証がなく、
軽減策 (RLHF 等) は 外部から 加える 必要がある。

**欠陥 6 : 自己説明能力の 欠如**

標準的な トランスフォーマーは 自己説明能力を 持たず、
なぜ その出力を 生成したかの 直接的な 説明が 困難。
解釈可能性研究は 進展しているが、
追加の 専門的 分析を 要する。

### 既存パッチ解決策の 限界

トランスフォーマーの 各欠陥に対して、
既存研究は 多様な 解決策を 提案してきた。
しかし これらは いずれも、
アーキテクチャの 外側から 欠陥を 補う パッチであり、
設計原理の 変更を 伴わない。

**O(n²) 問題への パッチ:**

Sparse Attention (Longformer, BigBird) は 注意の 範囲を 制限し、
Flash Attention は ハードウェアレベルの 最適化を 施す。
これらは 定数倍の 改善を もたらすが、
計算量の オーダーを 変えるものではない。
Mamba 等の SSM は アーキテクチャを 置き換えることで O(n) を 達成するが、
記憶や 学習の 問題は 残されたままである。

**記憶構造への パッチ:**

RAG は 外部データベースからの 検索を 可能にするが、
記憶は 人間が 設計・管理する 静的なものであり、
システムが 自律的に 「何を 覚えるべきか」を 判断する 機構を 持たない。
MemGPT は OS の 仮想メモリに 倣った 階層構造を 提案するが、
記憶の 移動ルールは 事前に 設計されたものであり、
経験から 学習されるものではない。
コンテキストウィンドウの 拡張 (100K〜1M tokens) は
作業記憶を 延長するに 過ぎず、
セッションをまたぐ 永続的 記憶とは 本質的に 異なる。

より 高度な アプローチとして、
Google Research の HOPE (Behrouz et al., Nested Learning, NeurIPS 2025) は
生物学的 神経可塑性を 模倣した 多タイムスケール 更新により
破滅的忘却を 解消し、
長文脈タスクを 含む 複数の 継続学習タスクで 有望な 性能を 示している。
HOPE は 継続学習において 現時点での 最先端と 位置づけられる。

しかし HOPE の 記憶は 数値的・連想的 であり、
なぜ その情報を 記憶したか、なぜ その応答が 有効だったかを
言語で 説明する 能力を 持たない。
また HOPE は 依然として 大規模 事前学習データを 前提とし、
ゼロから 言語を 獲得する 設計には なっていない。

**共通の 問題:**

これらの パッチは それぞれ 単独の 欠陥に 対処するが、
複数を 組み合わせた 場合の 相互作用が 保証されない。
RAG + EWC + Sparse Attention を 積み上げた システムは、
各コンポーネントの 複雑性が 乗算的に 増大し、
設計の 一貫性と 理論的 保証を 失う。

根本的な 問いは 次の 通りである:
なぜ これほどの パッチが 必要なのか?
それは、トランスフォーマーが
記憶・学習・適応を 設計原理として 持たない
アーキテクチャだからである。

### Contributions

**貢献 1 : 4層 統合アーキテクチャの 提案**

Mamba SSM (O(n) 連続処理) +
LaEMe (言語評価記憶) +
Active Inference / DEM (予測誤差学習) +
DGM (アーキテクチャ自己改善)
を 統合した アーキテクチャを 提案する。
この 組み合わせの 実装例は、
著者が 調査した 範囲では 先行研究に 存在しない。

**貢献 2 : 連続 SSM × Active Inference の 統合**

Mamba (連続状態空間) と
Active Inference を
連続時間 DEM を 接続層として 統合する 設計を 提案する。
Mamba の 状態遷移と Active Inference の 自由エネルギー最小化は
ともに 連続時間 ODE で 記述されるため、両者の 接続が 数学的に 自然である。
実装上の 制約から 離散化が 必要な 場合は、
意味的に 定義された 離散状態への 学習済み マッピング (方式 A) を
フォールバックとして 用いる。

**貢献 3 : LaEMe — 言語評価記憶**

HOPE (Behrouz et al., NeurIPS 2025) に 代表される
数値的・連想的 記憶とは 異なり、
LaEMe は 「なぜ 有効だったか」を 自然言語で 評価・保存する
評価的 記憶機構を 設計する。
これにより:
- 記憶が 解釈可能になる。
- 人間が 記憶の 内容に 直接 介入できる。
- 記憶の 質が 数値精度ではなく 言語的 妥当性で 管理される。

**貢献 4 : 継続的・自律的 知識獲得の 設計**

事前学習で 獲得した 言語能力を ブートストラップとし、
それ以降の 知識・評価基準の 獲得を
経験と LaEMe による 評価ループから
継続的・自律的に 行う 設計を 提案する。
言語の 初期獲得そのものを ゼロから 行うことは 主張しない。
獲得後の 継続学習を、
静的コーパスへの 再学習なしに 実現する点に 新規性がある。

**貢献 5 : パッチではなく 設計原理による 欠陥への 統合的 対処**

既存研究が トランスフォーマーの 各欠陥を
外部機構で 補ってきたのに対し、
本アーキテクチャは 6つの 欠陥への 対処を、
設計の 段階で カバーする。
各欠陥への 達成度は 一律ではなく、
完全な 解消・大幅な 軽減・軽減・部分的な 対処に 分かれる。
この 区別は Section 4.4 で 正直に 提示する。

## Section 2 : Background

### 2.1 State Space Models と Mamba

状態空間モデル (SSM) は、
隠れ状態 $h(t)$ を 介して 入力 $x(t)$ から 出力 $y(t)$ を 生成する 系として 定式化される。
連続時間では、状態の 時間発展は 線形 常微分方程式として 与えられる。

$$\dot{h}(t) = Ah(t) + Bx(t), \quad y(t) = Ch(t)$$

実装では これを ステップ幅で 離散化し、
再帰形式 $h_t = \bar{A} h_{t-1} + \bar{B} x_t,\ y_t = C h_t$ を 得る (4.1 参照)。

S4 (Gu et al., 2021) は この 線形 SSM を 深層学習に 適用し、
長系列処理における 並列計算を 実現した。
Mamba (Gu & Dao, 2023) は S4 を 発展させ、
$A, B, C$ を 入力の 関数とする 選択的状態空間を 導入した。
これにより モデルは 系列中の どの 情報を 状態に 保持するかを
入力に 応じて 動的に 決定できる。

推論は 再帰形式で O(n)、
訓練は Parallel Scan により 仕事量 O(n)・並列ステップ深さ O(log n) を 達成し、
標準的な self-attention の O(n²) を 原理的に 回避する。
また 固定サイズの 隠れ状態により、
理論上 無限の 系列長を 一定のメモリで 処理できる。

Mamba を 基盤とした 継続学習の 研究も 進展しており、
Lee et al. (2026) は 事例なしで SSM を 継続学習させる 手法を CVPR 2026 で 提案し、
Cheng et al. (2024) は Mamba の 事前学習済みモデルを
ヌル空間への 射影で 保護しながら 継続的に ファインチューニングする 手法 Mamba-CL を 示している。
本アーキテクチャは これらと 相補的であり、継続学習の 機構を 設計原理として 内包する 点で 異なる。

### 2.2 Active Inference と 自由エネルギー原理

自由エネルギー原理 (FEP; Friston, 2010) は、
知覚と 行動を 変分自由エネルギー $\mathcal{F}$ の 最小化として 統一的に 記述する。

$$\mathcal{F} = \mathbb{E}_q[\log q(s) - \log p(o, s)]$$

ここで $q(s)$ は 内部状態 $s$ に 関する 信念分布、
$p(o, s)$ は 観測 $o$ と 状態の 結合分布である。

連続時間 実装である DEM (Dynamic Expectation Maximization; Friston, 2008) は、
一般化座標上で $\mathcal{F}$ を 最小化し、
更新則を 常微分方程式として 表現する。
この 連続時間 定式化は Mamba の 連続状態空間と
ともに 連続時間 ODE で 記述されるため、接続が 数学的に 自然であり、
SSM を 生成モデルとして 採用できる。

Active Inference を 高度な AI への 経路として 位置づける 議論 (Maier, 2025) や、
離散モデル上での 構造学習・推論への 拡張 (Friston et al., 2025) など、
本枠組みは 近年も 活発に 展開されている。

### 2.3 LaEMe

LaEMe (Language-based Evaluative Metadata for Reflective Memory Systems) は、
任意の 経験・知覚・相互作用に対して 自然言語の メタデータを 付与・保存・活用する
汎用 フレームワークである。

従来の 数値的 記憶は、評価の 根拠を 数値空間に 圧縮するため 解釈可能性を 失う。
LaEMe は 言語空間を 記憶の 基盤とすることで、3つの 性質を 実現する。

1. **解釈可能性**: 「なぜ その応答が 有効だったか」を 人間が 直接 読める。
2. **介入可能性**: 人間が 記憶の 内容を 直接 修正・削除できる。
3. **生成的 利用**: 蓄積された 言語メタデータから 新しい 応答候補を 生成できる。

LaEMe が 対象とする 記憶の 種類は「評価理由」に 限らない。
応答への 評価 (「この 返答は 正確だが 共感が 不足していた」) のみならず、
知覚の 説明 (「入力 X は 感情的な トーンを 含む」)、
環境の 観察 (「対話相手は 現在 短い 応答を 好む 傾向にある」)、
および 抽象化された 行動原則まで、すべてを 言語で 記録できる。
この 汎用性が、LaEMe を 単なる 評価ログではなく
反省的記憶 (reflective memory) の 基盤たらしめる 核心的 特性である。

**MARS との 関係:**
Hou et al. (2026) の MARS は、インタラクションごとに
「なぜ 有効/無効 だったかの 抽象原則 (principle)」と
「次回 同様の 状況で 取るべき 行動 (procedure)」の 2種の 振り返りメモを 生成し、
外部フィードバックなしに 自己改善を 実現する。
LaEMe と MARS の 関係は 包含関係として 整理できる。
MARS が 定義する principle / procedure メモは、LaEMe エントリの 特殊形であり、
LaEMe は MARS を 内部に 含む より 広いフレームワークである。
本アーキテクチャでは、LaEMe の 各エントリを 以下の 構造で 定義する。

```
{
  context    : 当該インタラクションの 文脈,
  response   : 生成した 応答,
  evaluation : 評価テキスト (なぜ 有効/無効 だったか),
  principle  : 抽象化された 行動原則 (MARS principle に 対応),
  procedure  : 次回 同様状況での 推奨行動 (MARS procedure に 対応)
}
```

**他の 記憶システムとの 比較:**
LaEMe は HOPE (Behrouz et al., NeurIPS 2025) 等の
数値的 記憶システムと 対立するものではなく、
数値的 処理層の 上位に 位置する 言語的 評価層として 設計できる。
また EM-LLM (Nguyen et al., 2024) は Bayesian surprise による
エピソード境界検出で 10M tokens を 効率処理するが、
LaEMe との 本質的 差異は 記憶の 内容にある。
EM-LLM は「何が 起きたか」というエピソード事実を 覚えるのに対し、
LaEMe は「なぜ そうすべきだったか」という 評価理由と 原則を 保存する。
さらに Jaiswal et al. (2026) の MemoryLLM は フィードフォワード層の 重みを
トークン単位の 記憶として 再構成するが、
記憶が 重みに 暗黙的に 埋め込まれるため 人間の 介入が 難しい。
LaEMe の 介入可能性は この 点で 明確に 区別される。
Wu et al. (2025) は LLM 時代の 記憶機構を 人間の 記憶モデルと 対応づけた
包括的 サーベイを 提供しており、LaEMe の 位置づけを 理解する 文脈として 有用である。

### 2.4 Evolution Strategies (ES)

ES は 勾配を 必要としない ブラックボックス 最適化手法であり、
パラメータ分布 $\pi(\theta | \psi)$ から サンプル $\theta_i$ を 生成し、
評価値 $F(\theta_i)$ に 基づいて 分布 $\psi$ を 更新する。

Natural Evolution Strategies (NES; Wierstra et al., 2014) は
自然勾配方向に $\psi$ を 更新することで 安定性を 高める。
近年は、勾配を 用いない ES と 勾配ベースの GRPO が
同等の 精度に 達しながら パラメータ空間で 異なる 幾何を 描くことが 報告されており (Hoy et al., 2026)、
LLM の 事後学習における ES の 再評価が 進んでいる。

本論文では ES の 独立実装の 要否を 設計判断として 扱い (3.2.1 参照)、
その 役割を Layer 3 (Active Inference) に 吸収できる 可能性を 示す。

### 2.5 Darwin Gödel Machine (DGM)

DGM (Zhang et al., 2025) は、
自己改変 (Gödel Machine; Schmidhuber, 2003) と
進化的探索を 統合した 自律改善システムである。

Lavi における DGM は Layer 4 として 位置づけられ、
Layer 1〜3 が 生成した 評価データを 基に、
アーキテクチャ自体を 改善する 役割を 担う。
これは 重みの 更新を 超えた、
アーキテクチャレベルの 自己改善である。

自己改善の 関連研究として、Hou et al. (2026) の MARS は
メタ認知的な 振り返り (原理ベースと 手続きベースの 2種) を 用いて
1サイクル内で 自己進化を 実現する 手法を 提案しており、
外部フィードバックへの 依存を 最小化する 点で 本研究と 問題意識を 共有する。
MARS の principle / procedure 構造は LaEMe エントリの 特殊形として 統合される (2.3 参照)。
Lavi の DGM は 重みを 超えた アーキテクチャ自体の 改変を 目指す 点で MARS と 相補的である。

また、Chen et al. (2025) は 自己改善を Generator-Verifier-Updater 構造として
定式化した Self-Play Framework を 提案しており、
重みの 変更を 要さずに 自己改善ループを 成立させる 設計であり、
DGM が 未成熟な 段階 (Phase 1) での 軽量 L4 として 機能させることができる。
本アーキテクチャでは L4 を 以下の 段階で 発展させる。

```
Phase 1 : Self-Play (Generator-Verifier-Updater)  ← 重み変更なし、即時 実装可能
Phase 2 : DGM への 移行          ← 評価データが 十分に 蓄積された後
```

## Section 3 : Proposed Architecture

### 3.1 アーキテクチャ概要

```
┌─────────────────────────────────────────┐
│  Layer 1 : Mamba SSM                    │
│    ← 応答生成・自己評価・変換指示を 兼任 │
├─────────────────────────────────────────┤
│  Layer 2 : LaEMe                        │
│    ← 評価・保存・候補生成を 担う         │
├─────────────────────────────────────────┤
│  Layer 3 : Active Inference (DEM)       │
│    ← LaEMe 評価を 観測空間として 統合    │
├─────────────────────────────────────────┤
│  Layer 4 : DGM                          │
│    ← アーキテクチャ自己改善             │
└─────────────────────────────────────────┘
```

Mamba は 単一のモデルでありながら、
文脈に 応じて 以下の 複数の 役割を 担うことができる。

- **応答生成**: 会話文脈から 次の 発話を 生成する。
- **自己評価**: (文脈 + 応答 + 外部反応) から 評価文を 生成する。
- **変換指示**: 評価文から Mamba 更新の 方向を 言語で 提案する。

基盤モデルとして **Mamba2 事前学習済みモデル** (例: `nvidia/mamba2-8b-3t-4k`) を 採用する。
Mamba2 は Mamba の 発展形であり、SSD (State Space Duality) フレームワークによる 効率的な 並列計算を 実現している。

#### 行動空間の 段階的 定義

Lavi の 行動空間は 以下の 順序で 拡張される。

```
Phase 1 : 発話      ← Cataa を通じた Loa との 自律的な 会話
Phase 2 : 検索      ← インターネット検索を Active Inference の 行動として 実行
Phase 3 : 自己改良  ← DGM による アーキテクチャ自己改善
```

Active Inference における 「行動」は 自由エネルギーを 最小化する 手段として 定義される。
Phase 1〜3 のすべての 行動は この 統一的な フレームで 記述できる。

### 3.2 設計判断の 記録

#### 3.2.1 ES の 要否と 統合判断

**原案:** ES を 独立した 層として 設け、
LaEMe の スカラー評価値で 応答候補の 重みを 更新する。

**問題 1 — スカラー変換の 情報損失:**
LaEMe の 評価は 本質的に 言語的であり、
スカラーへの 圧縮は 「なぜ 有効だったか」という
評価の 核心的な 情報を 捨てることになる。

**問題 2 — 重複の 可能性:**
LaEMe の 評価文を Layer 3 (Active Inference) の 観測空間として
直接 統合できる場合、
Lavi が 予測した 評価と 実際の 評価との 差分が
予測誤差として 機能し、
Layer 1 (Mamba) の パラメータが 更新される。
これは ES が 達成しようとした 重み更新と
機能的に 等価である。

**結論:**
Active Inference が LaEMe 評価を 観測として 処理できる場合、
ES は 独立した 層として 不要になる。
本アーキテクチャでは ES を 独立した 層として 設けず、
その 役割を Layer 3 (Active Inference) に 吸収する。
独立実装が 有効かどうかは 今後の 検証課題とする。

#### 3.2.2 連続 × 離散 SSM の 接続方式

**方式 A : 階層分離**
Mamba (連続) の 出力を softmax で 離散化し、
事前に 意味的に 定義した 離散状態に マッピングした上で
Active Inference (離散) に 渡す。
> 利点: 実装が 比較的 容易。
> 欠点: 離散化による 情報損失、および 離散状態の 事前設計が 必要。

**方式 B : 連続時間 Active Inference (DEM)**
Friston の DEM は 連続時間上で 変分自由エネルギーを 最小化し、
Mamba の 連続状態空間と ともに 連続時間 ODE で 記述されるため、
離散化なしに 自然に 接続できる。
> 利点: 情報損失なし、理論的 一貫性が 高い。
> 欠点: 実装難度が 高い。

**方式 C : 混合状態空間**
> 利点: 最も 表現力が 高い。
> 欠点: 設計・実装ともに 最も 困難。

**採用:** 理論的 合理性から 方式 B を 主軸とする。
実装上の 制約が 生じた 場合は 方式 A を フォールバックとし、
方式 C は 将来の 拡張として 位置づける。

**L3 実装の フォールバック階層:**
方式 B (DEM) の 実装が 困難な 場合に 備え、L3 の 実装候補を 明示的に 階層化する。

```
第1候補 : DEM (連続時間 Active Inference)   ← 理論的 一貫性 最高
第2候補 : Predictive Coding (PC)            ← DEM と 同じ 予測誤差の 枠組み、実装が 容易
第3候補 : Fast Weight Programmer (FWP)      ← 線形 Transformer と 等価、RNN で 実装可能
```

Lauber et al. (2025) は PC が 継続学習で バックプロパゲーションを 上回る 効率を 示すことを 報告しており、
第2候補としての 実用性は 高い。
Schlag et al. (2025) の FWP は、外部信号で 高速重みを 書き換える RNN として
線形 Transformer および Neural ODE と 数学的に 等価であることが 示されており、
DEM の 連続時間 定式化への 近似として 採用できる。

#### 3.2.3 評価・変換機構の 設計

**案 i : スカラー変換**
> 棄却理由: 言語の 意味的 豊かさを スカラーに 圧縮することで、
> 評価の 本質的 情報が 失われる。

**案 ii : 3モデル 分離**
> 棄却理由: Mamba が 応答を 生成できるならば、
> 同じ Mamba が 評価文・変換指示も 生成できる。

**採用 : Mamba 一体化**
Mamba が 文脈に 応じて 応答生成・自己評価・変換指示を 兼任する。
これにより スカラー変換を 経由せず、
言語空間のまま 学習ループを 完結できる。

### 3.3 オンライン学習の 設計空間

| 案 | 概要 | 利点 | 欠点 |
|---|---|---|---|
| フルバックプロパゲーション | 全重みを 勾配で 更新 | 表現力 最大 | 計算コスト 最大 |
| Fast Weights | 高速重みと 低速重みの 2層 | Active Inference の 2タイムスケールと 整合 | 設計が 複雑 |
| LoRA オンライン | 低ランク アダプタのみ 更新 | 安定・低コスト | 表現力に 上限 |
| 純粋文脈学習 | 重み更新なし、LaEMe を 文脈に 含める | コスト最小、即時 実装可能 | 真の 重み更新でない |
| Hebbian + 変調 | LaEMe が 更新対象の 局所重みを 制御 | 生物的・低コスト | 表現力 限定 |
| リザーバー | 出力層のみ 更新 | 最安価 | 表現力 最低 |

**採用方針 (段階的):**

```
Step 1 : 純粋文脈学習 (即時 実装可能)
Step 2 : Fast Weights (LaEMe が 成長した後)
Step 3 : DGM による 自動改善 (アーキテクチャ成熟後)
```

### 3.4 Bootstrap 問題と 外部評価者の 役割

初期の Lavi は 語彙も 自己評価能力も 持たない。
**主評価者は Loa** (Claude Code CLI エージェント) であり、
著者 Gles は 補助的な 役割 (オプション) を 担う。
基本設計として、Loa と Lavi の 2者間で 学習ループが 自律的に 回ることを 目指す。

```
初期フェーズ : Loa が 主評価者として すべての 応答を 評価する
              (Gles はオプションで 補助)
  → LaEMe に (文脈, 応答, 評価) が 蓄積される
      ↓
中期フェーズ : Mamba が 過去評価を 文脈として 参照し、
              自己評価を 生成し始める
      ↓
成熟フェーズ : Mamba が 自律的に 自己評価できるようになる
  → Loa の 介入頻度を 段階的に 下げる
```

外部評価者の 役割は 「正解を 与える」ことではなく、
「Lavi が 評価基準を 構築するための 素材を 提供する」ことである。

**MARS による Bootstrap 加速:**
MARS (Hou et al., 2026) の principle / procedure 構造を LaEMe に 統合することで、
Bootstrap フェーズの 短縮を 設計的に 意図する。
Loa が 評価を 与える 際、その 評価から 自動的に
`principle` (「この 場面では 誠実さを 共感より 優先すべき」等の 抽象原則) と
`procedure` (「次回 同様の 状況では まず 事実確認を 行ってから 感情的 応答を 返す」等) を
抽出し LaEMe に 保存する。
中期フェーズ以降、Mamba は この principle / procedure を 文脈として 参照し、
Loa への 依存なしに 自己評価を 生成できるように 漸進的に 移行する。
これにより Bootstrap 設計の 弱点であった「外部評価者 依存」が 構造的に 緩和される。

## Section 4 : Analysis

本章では、Section 1 で 列挙した 6つの 欠陥に対し、
提案アーキテクチャが 各欠陥を どの 程度・どの 機構で 扱うかを 分析する。
分析の 中心は 3点である。
第1に O(n²) 複雑度の 解消が 理論的に 成立する 根拠 (4.1)、
第2に LaEMe 評価を Active Inference に 渡す 変換設計の 精密化 (4.2)、
第3に 事前学習済みモデルを 基盤としつつ 継続的 知識獲得を 実現する 設計の 妥当性 (4.3) である。
最後に、6欠陥への 達成度を 過大評価せず 正直に 区別する (4.4)。

### 4.1 O(n²) 複雑度の 解消 — 理論的 分析

標準的な self-attention は、
系列長 $n$ の 各トークンが 他の すべての トークンに 注意を 向けるため、
$n \times n$ の 注意行列を 構成する。
計算量・メモリともに $O(n^2)$ であり、
これは 定数倍の 最適化では 取り除けない 漸近的 オーダーである。

提案アーキテクチャは この欠陥を、
**Mamba2 を 基盤に 採用することで 解消する**。
ただし この 解消は Mamba (Gu & Dao, 2023) および Mamba2 (Dao & Gu, 2024) から 継承したものであり、
本研究 独自の 貢献ではない。
本研究の 新規性は この $O(n)$ 基盤を
言語評価記憶・Active Inference と 統合する 点 (貢献 1) に ある。

解消が 成立する 根拠は 状態の 有限性に ある。
SSM は 再帰形式

$$h_t = \bar{A} h_{t-1} + \bar{B} x_t, \quad y_t = C h_t$$

において、過去の 全情報を **固定次元の 隠れ状態** $h_t \in \mathbb{R}^{d_\text{state}}$ に 圧縮する。
1ステップあたりの コストは $O(d_\text{state} \cdot d_\text{model})$ であり、
系列長 $n$ に 依存しない。
したがって 推論全体は $O(n)$、
かつ $n \times n$ の 相互作用行列を 一度も 実体化しない。
これが オーダー自体を 変える 根拠である。

訓練時には、Mamba2 の SSD (State Space Duality; Dao & Gu, 2024) により、
SSM の 計算が **構造化行列 (半分離行列) の 積**として 書き直せる。
これにより、選択的 SSM の 表現力を 保ちつつ、
行列積に 最適化された ハードウェア上で 効率的な 並列訓練が 可能になる。

**正直な 留保 (状態容量の トレードオフ):**
$O(n)$ 化の 代償として、固定サイズの $h_t$ は
無限長の 文脈を 無損失で 保持できない。
これは SSM 一般に 知られる 想起と 効率の トレードオフであり、
選択的 SSM (入力依存の $\bar{A}, \bar{B}, C$) は これを 緩和するが 消去はしない。
この 留保は 本研究において むしろ **設計上の 必然性**を 与える。
すなわち、有限の 再帰状態を 作業記憶として 用い、
セッションを またぐ 長期記憶は 外部の LaEMe (4.2, 欠陥 2) に 担わせる。
$O(n)$ 基盤と 永続的 言語記憶の 分業は、
状態容量の トレードオフへの 構造的 応答に なっている。

### 4.2 LaEMe → 埋め込み → Active Inference の 変換設計

LaEMe 評価を Active Inference に 渡す 際の 設計上の 要点は、
評価を スカラー (1次元) に 圧縮せず、
意味を 保つ 高次元 埋め込みベクトルとして 渡すことである。

**なぜ ベクトルでなければ ならないか:**
Active Inference は 変分自由エネルギー $\mathcal{F}$ を 最小化する 枠組みであり、
$\mathcal{F}$ を 定義する 以上、観測 $o$ は ベクトル空間の 元でなければならない。
離散トークン列の ままでは $\mathcal{F}$ を 定義できないため、
評価テキストを 埋め込みベクトルへ 写像する 段階は 必然的に 入る。
要点は その 写像で 意味の 次元を 落とさないこと、
すなわち スカラーへの 圧縮を 避けることに ある。

**変換設計:**
LaEMe が 保存する 評価は 自然言語の テキスト $e$ である。
これを Active Inference の 観測として 用いるため、
**同一の Mamba2 モデルを 符号化器として 流用**し、
$e$ を 埋め込みベクトル $\phi(e) \in \mathbb{R}^{d}$ に 写像する。
ここで Mamba2 を 符号化器に 流用するのは、
評価文・変換指示を Mamba が 兼任するという 一体化設計 (3.2.3) と 整合する。

観測を $o = \phi(e)$ と し、
Lavi は 内部状態 $s$ から 期待される 評価埋め込み $\hat{o} = g(s)$ を 予測する。
予測誤差は 埋め込み空間上で

$$\varepsilon = \phi(e) - \hat{o}$$

と 定義され、$\mathcal{F}$ の 最小化が この $\varepsilon$ に 沿って
Layer 1 (Mamba) の パラメータを 更新する。
連続時間 実装 (DEM, 方式 B) では、
$\phi(e(t))$ を 一般化座標上の 軌道として 扱い、
$\mathcal{F}$ への 勾配流で 更新する。

**なぜ スカラーでは 不十分か (情報損失の 具体化):**
スカラースコアは、
たとえば「応答は 事実誤りだが 共感的」と「応答は 正確だが 冷淡」を
同一の 数値に 潰しうる。
この とき 更新方向は「良し悪しの 大きさ」しか 運ばない。
高次元 埋め込み $\phi(e)$ は 両者を 別方向の ベクトルとして 区別するため、
予測誤差 $\varepsilon$ が **「なぜ そう 評価されたか」という 方向情報**を 保持する。
これが スカラー圧縮を 避ける 核心的 理由であり、
LaEMe の 解釈可能性 (貢献 3) を 学習信号の 段階まで 一貫させる。

### 4.3 継続的 知識獲得の 実現可能性

提案システムは 基盤に Mamba2 事前学習済みモデルを 採用する。
したがって 本研究は **言語の 初期獲得を 担うのではなく、獲得後の 継続的 知識獲得を 担う**。
事前学習済みの 重みは 構文・基礎的 意味を 与える ブートストラップであり、
本研究が ゼロから 立ち上げるのは
**ドメイン知識と 評価基準**である ($t=0$ で LaEMe は 空)。
この 区別を 明確にすることで、欠陥 4 への 主張を 防御可能な 範囲に 限定する。

**実現可能性の 段階的 論証:**

*Step 1 — 純粋文脈学習 (即時 実現可能):*
LaEMe の エントリを 文脈に 注入し、重み更新なしで 応答に 反映する。
Mamba の 再帰状態は 長文脈を $O(n)$ で 処理できるため、
蓄積した 評価を 文脈として 与える コストは 系列長に 線形である。
この 段階は 追加の 学習機構を 要さず、本日の 時点で 実装可能である。

*Step 2 — Fast Weights (LaEMe 成熟後):*
高速重みと 低速重みの 2タイムスケール (Schmidhuber, 1992; Ba et al., 2016) を 導入し、
LaEMe 評価で 変調された 高速重みの 更新により、
文脈窓を 超える 持続的 適応を 実現する。
Fast Weights は 既存研究で 確立された 機構であり、
本設計は その 変調信号として LaEMe を 用いる 点が 新しい。

*Step 3 — DGM (アーキテクチャ成熟後):*
Layer 4 が 蓄積評価を 基に アーキテクチャ自体を 改善する。

**破滅的忘却への 構造的 応答:**
継続学習の 主要リスクである 破滅的忘却に対し、
本アーキテクチャは 3つの 構造的 緩和を 持つ。
第1に、獲得知識を 解釈可能な 外部記憶 LaEMe に オフロードするため、
知識が 重みのみに 宿らない。
第2に、Active Inference の 更新は 全データへの フルバックプロパゲーションではなく、
予測誤差で 変調された 局所的 更新であり、
過去表現の 大域的 上書きを 起こしにくい。
第3に、Gradient Episodic Memory (Lopez-Paz & Ranzato, 2017) の 設計思想を 参照し、
LaEMe に 保存された 過去インタラクションの 評価を「勾配制約の 素材」として
Active Inference の 更新に 組み込むことで、
過去応答への 性能劣化を 抑制する 補完的 機構として 機能させられる。

### 4.4 欠陥への 達成度 — 正直な 区別

貢献 5 で 述べた 通り、6欠陥への 達成度は 一律ではない。
過大評価を 避けるため、各欠陥を
**解消 / 大幅軽減 / 軽減 / 部分対処**に 区別して 提示する。

| 欠陥 | 主たる 機構 | 達成度 | 留保 |
|---|---|---|---|
| 1 O(n²) 複雑度 | Mamba2 (SSD) | **解消** | Lavi 独自でなく 継承。状態容量の トレードオフあり |
| 2 記憶構造の 欠如 | LaEMe 永続記憶 | **解消** | 記憶を 設計原理に 内蔵 |
| 3 重みの 準固定性 | Active Inference リアルタイム更新 | **大幅軽減** | Step1→2→3 の 段階達成。完全自動化は 未検証 |
| 4 事前学習依存 | 継続学習として 再定義 | **部分対処** | 言語の 初期獲得は 依存。以降の 知識獲得は 自律 |
| 5 ハルシネーション | LaEMe 評価フィルタ | **軽減** | 解消ではない。未評価の 新規状況では 残存 |
| 6 自己説明能力 | LaEMe 評価理由の 言語保存 | **大幅軽減** | 評価理由は 説明可能。生成過程 自体の 説明とは 区別 |

特に 欠陥 5・6 について 注意を 促す。
欠陥 5 では、LaEMe は 評価済み応答の 信頼性を 高めるが、
未評価の 新規状況での 生成を 保証しない。
これは パッチ的な RLHF より 構造的だが、解消ではなく 軽減である。
欠陥 6 では、LaEMe は「なぜ その応答が **有効だったか** (事後評価)」を 言語化するが、
「なぜ その応答を **生成したか** (生成過程そのもの)」の 機構的 説明とは 異なる。
両者を 区別することで、自己説明能力の 主張を 検証可能な 範囲に 留める。

## Section 5 : Discussion

Section 4 では 提案アーキテクチャが 各欠陥を どう 扱うかを 分析した。
本章では、設計を 実際に 動かす 段階で 直面する 制約と、
現時点で 未証明の 要素を 正直に 議論する。
過大評価を 避ける 姿勢 (4.4) を Discussion 全体に 一貫させる。

### 5.1 評価ループの スループット制約

本アーキテクチャの 学習は、外部評価者 Loa が 応答を 評価し、
その 評価を LaEMe に 蓄積する ループに 依存する (3.4)。
ここで Loa は **Claude Code CLI エージェント** (Opus / Sonnet) であり、
Cataa が `claude` コマンドを サブプロセスとして 起動して 駆動する。
従量制の API ではないため、束縛条件は
「インタラクションあたりの 金額」ではなく、
**CLI 駆動の スループットと レート制限**である。

継続学習は 多数の インタラクションを 必要とするため、
評価者を ループに 含む 構成では 評価の 速度が 学習の 律速段階になる。
これに対し 本設計は 3つの 緩和を 持つ。
第1に、ブートストラップ後に 自己評価へ 移行し (3.4 成熟フェーズ)、
外部評価への 依存頻度を 段階的に 下げる。
第2に、Step 1 の 純粋文脈学習は 重み更新を 伴わないため、
1イテレーションの コストが 小さく、試行回数を 稼げる。
第3に、評価を 非同期・バッチ化することで、
対話の リアルタイム性と 学習の スループットを 分離できる。
初期段階の ボトルネックは 計算資源ではなく 評価スループットである、
という 認識が 実験計画上 重要である。

### 5.2 状態容量の トレードオフの 実証課題

4.1 で 述べた 通り、固定サイズの 再帰状態 $h_t$ は
無限長の 文脈を 無損失で 保持できない。
本設計は 長期記憶を 外部の LaEMe に 担わせることで これに 応答するが、
LaEMe からの 検索・文脈注入にも 固有の 限界がある。
注入できる 評価量は 文脈窓に 制約され、
どの 評価を 注入するかの 選択自体が 新たな 設計問題になる。

これは 理論だけでは 決着せず、実証を 要する。
具体的な 実験として、
(1) 対話長に対する 想起精度の 劣化を 計測し、
(2) LaEMe 注入が その 劣化を どの 程度 回復するかを 測る。
この 2点が、$O(n)$ 基盤と 外部記憶の 分業が 実際に 機能するかの
最初の 検証ポイントになる。

### 5.3 未証明の 設計要素

本アーキテクチャには、理論的 合理性から 採用しながらも、
実装・実験では 未検証の まま 残る 要素が 複数 ある。
査読者の 判断に 資するため、これらを 明示する。

- **連続時間 統合 (方式 B, DEM)**: 理論的 一貫性は 高いが (3.2.2)、
  実装難度も 高く、実現可能性は 未証明である。
  方式 A (離散化フォールバック) が 必要になる 可能性がある。
- **破滅的忘却**: 4.3 の 構造的 緩和は あくまで 設計上の 論証であり、
  実際に 忘却が 抑制されるかは 実験で 示す 必要がある。
- **Layer 4 (DGM)**: 4層の 中で 最も 未成熟であり、
  アーキテクチャ自己改善は 現時点で 構想段階に とどまる。
- **ゼロからの 評価基準獲得**: 継続学習としての 再定義 (4.3) は
  主張を 防御可能にしたが、評価基準が 経験のみから
  実際に 立ち上がるかは 実証されていない。

これらを 隠さず 述べることは 本研究の 弱点ではなく、
検証すべき 仮説を 明確にする 作業である。

### 5.4 実装ステータスと ロードマップ

誠実さのため、実装の 現状を 明記する。
本稿 執筆時点で Lavi コアは 設計段階にあり、
Cataa (Gles / Lavi / Loa の 3者対話 CLI) を 評価基盤として 先行整備している。

実装言語は Mojo を 採用する (3.5.1)。
Mamba2 と Active Inference の Mojo 実装は 現存しないため、
本実装自体が エコシステムへの 貢献となる 一方、
前例のなさは 実装リスクでもある。
詰まった 場合の フォールバックは Rust とする。

行動空間は 段階的に 拡張する (3.1)。
Phase 1 (発話) を Cataa 上の Loa との 自律対話で 立ち上げ、
Phase 2 (検索)、Phase 3 (自己改良) へ 順に 拡げる。
Stage 1 の 実行環境は Google Colab 無料枠とし、
規模が 確定した 段階で 再評価する (3.5.2)。

### 5.5 反証可能性

科学的 主張として、本設計を **否定しうる 観測**を 先に 述べる。
以下の いずれかが 成り立つ場合、対応する 設計仮説は 棄却される。

- LaEMe の 埋め込みが Active Inference の 予測誤差を
  有意に 減らせない場合、言語評価記憶と 予測符号化の 統合 (4.2) は 失敗である。
- 蓄積した LaEMe 評価から Mamba が 自己評価を 学習できない場合、
  ブートストラップ設計 (3.4) は 失敗である。
- 外部記憶を 持つにもかかわらず 継続更新が 破滅的忘却を 起こす場合、
  継続学習の 主張 (4.3) は 失敗である。

これらの 反証条件を 事前に 定義することで、
本研究を 検証可能な 科学的 提案として 位置づける。

## Section 6 : Conclusion

本研究は、トランスフォーマーが 抱える 6つの 構造的 欠陥
(O(n²) 複雑度、記憶構造の 欠如、重みの 準固定性、事前学習依存、
ハルシネーション、自己説明能力の 欠如) を、
外部からの パッチではなく 設計原理の 段階で 扱う アーキテクチャ Lavi を 提案した。

Lavi は 4層から 成る。
Mamba2 SSM (O(n) 連続処理)、LaEMe (言語評価記憶)、
Active Inference / DEM (予測誤差による 統合学習)、
DGM (アーキテクチャ自己改善) である。
中心的な 設計判断は 2点である。
第1に、評価を スカラーに 圧縮せず 高次元 埋め込みとして Active Inference に 渡すことで、
言語評価の 意味構造を 学習信号の 段階まで 一貫させた (4.2)。
第2に、ゼロスタート 言語獲得という 当初構想を、
事前学習済み Mamba2 を ブートストラップとした「獲得後の 継続的 知識獲得」へ 再定義し、
主張を 防御可能な 範囲に 収めた (4.3)。

本研究は 達成度を 誇張しない。
6欠陥のうち、O(n²) と 記憶構造は 設計原理で 解消され、
重み固定と 自己説明は 大幅軽減、事前学習依存は 部分対処、
ハルシネーションは 軽減に とどまる (4.4)。
方式 B (DEM) の 実装可能性、破滅的忘却の 抑制、
評価基準の ゼロからの 獲得は、いずれも 今後の 実証課題である (5.3)。

今後の 作業は 3段階である。
第1に、純粋文脈学習による Phase 1 (発話) を Cataa 上で 立ち上げ、
状態容量と 想起精度の 関係を 計測する (5.2)。
第2に、Fast Weights へ 移行し、文脈窓を 超える 持続的 適応を 検証する。
第3に、DGM による アーキテクチャ自己改善を 導入する。
再現性のため、Phase 1 の 成果を Zenodo 等で 公開することを 検討する。

本アーキテクチャの 設計指針は 単純である。
最初から 高度な 能力を 持つ 必要はなく、
記憶・学習・適応を 設計原理として 備えていれば、
経験と 評価から 漸進的に 成長できる。
この 仮説を 反証可能な 形で 提示したことが、本研究の 核心的な 貢献である。
