## Section 1 : Introduction

### トランスフォーマーの 欠陥 (重要度順)

**欠陥 1 : O(n²) 処理複雑度**

標準的な self-attention は O(n²) であり、
長系列処理の 根本的な ボトルネックになる。
Sparse Attention 等の 変種で 緩和できるが、
設計原理の 変更を 伴う。

**欠陥 2 : 記憶構造の 欠如**

継続学習・生涯学習の 実装は 可能だが、
EWC・リプレイバッファ等の 追加機構が 必須となり、
アーキテクチャに 記憶が 設計されていないことによる
構造的 非効率が 残る。

**欠陥 3 : 重みの 準固定性**

重みの 持続的 更新は ファインチューニングを 要し、
コストが 高い。
インコンテキスト学習は セッション内の 一時的 適応に 過ぎず、
永続しない。

**欠陥 4 : 事前学習データへの 構造的 依存**

大量の 事前学習データへの 依存度が 構造的に 高く、
経験だけから 自律的に 言語を 獲得する 設計に なっていない。

**欠陥 5 : ハルシネーション**

生成機構 自体に 意味的 正確性の 保証がなく、
軽減策 (RLHF 等) は 外部から 加える 必要がある。

**欠陥 6 : 自己説明能力の 欠如**

標準的な トランスフォーマーは 自己説明能力を 持たず、
なぜ その出力を 生成したかの 直接的な 説明が 困難。
解釈可能性研究は 進展しているが、
追加の 専門的 分析を 要する。

### 既存パッチ解決策の 限界

トランスフォーマーの 各欠陥に対して、
既存研究は 多様な 解決策を 提案してきた。
しかし これらは いずれも、
アーキテクチャの 外側から 欠陥を 補う パッチであり、
設計原理の 変更を 伴わない。

**O(n²) 問題への パッチ:**

Sparse Attention (Longformer, BigBird) は 注意の 範囲を 制限し、
Flash Attention は ハードウェアレベルの 最適化を 施す。
これらは 定数倍の 改善を もたらすが、
計算量の オーダーを 変えるものではない。
Mamba 等の SSM は アーキテクチャを 置き換えることで O(n) を 達成するが、
記憶や 学習の 問題は 残されたままである。

**記憶構造への パッチ:**

RAG は 外部データベースからの 検索を 可能にするが、
記憶は 人間が 設計・管理する 静的なものであり、
システムが 自律的に 「何を 覚えるべきか」を 判断する 機構を 持たない。
MemGPT は OS の 仮想メモリに 倣った 階層構造を 提案するが、
記憶の 移動ルールは 事前に 設計されたものであり、
経験から 学習されるものではない。
コンテキストウィンドウの 拡張 (100K〜1M tokens) は
作業記憶を 延長するに 過ぎず、
セッションをまたぐ 永続的 記憶とは 本質的に 異なる。

より 高度な アプローチとして、
Google Research の HOPE (Nested Learning, NeurIPS 2025) は
生物学的 神経可塑性を 模倣した 多タイムスケール 更新により
破滅的忘却を 解消し、
長文脈タスクで Mamba2 や Titans を 上回る 性能を 示している。
HOPE は 継続学習において 現時点での 最先端と 位置づけられる。

しかし HOPE の 記憶は 数値的・連想的 であり、
なぜ その情報を 記憶したか、なぜ その応答が 有効だったかを
言語で 説明する 能力を 持たない。
また HOPE は 依然として 大規模 事前学習データを 前提とし、
ゼロから 言語を 獲得する 設計には なっていない。

**共通の 問題:**

これらの パッチは それぞれ 単独の 欠陥に 対処するが、
複数を 組み合わせた 場合の 相互作用が 保証されない。
RAG + EWC + Sparse Attention を 積み上げた システムは、
各コンポーネントの 複雑性が 乗算的に 増大し、
設計の 一貫性と 理論的 保証を 失う。

根本的な 問いは 次の 通りである:
なぜ これほどの パッチが 必要なのか?
それは、トランスフォーマーが
記憶・学習・適応を 設計原理として 持たない
アーキテクチャだからである。

### Contributions

**貢献 1 : 4層 統合アーキテクチャの 提案**

Mamba SSM (O(n) 連続処理) +
Active Inference / DEM (予測誤差学習) +
LaEMe (言語評価記憶) +
DGM (アーキテクチャ自己改善)
を 統合した アーキテクチャを 提案する。
この 組み合わせの 実装例は、
著者が 調査した 範囲では 先行研究に 存在しない。

**貢献 2 : 連続 SSM × Active Inference の 統合**

Mamba (連続状態空間) と
Active Inference を
連続時間 DEM を 接続層として 統合する 設計を 提案する。
Mamba の 状態遷移と Active Inference の 自由エネルギー最小化が
同一の 連続時間 フレームワーク上で 統合される。
実装上の 制約から 離散化が 必要な 場合は、
意味的に 定義された 離散状態への 学習済み マッピング (方式 A) を
フォールバックとして 用いる。

**貢献 3 : LaEMe — 言語評価記憶**

HOPE (NeurIPS 2025) に 代表される
数値的・連想的 記憶とは 異なり、
LaEMe は 「なぜ 有効だったか」を 自然言語で 評価・保存する
評価的 記憶機構を 設計できる。
これにより:
- 記憶が 解釈可能になる。
- 人間が 記憶の 内容に 直接 介入できる。
- 記憶の 質が 数値精度ではなく 言語的 妥当性で 管理される。

**貢献 4 : ゼロスタート 言語獲得の 設計**

大規模 事前学習データを 前提とせず、
模倣と LaEMe による 評価ループから
言語能力を 漸進的に 獲得する 設計を 提案する。

**貢献 5 : パッチではなく 設計原理による 欠陥解消**

既存研究が トランスフォーマーの 各欠陥を
外部機構で 補ってきたのに対し、
本アーキテクチャは 6つの 欠陥すべてを
設計原理の 段階で 解消することを 目指す。

## Section 2 : Background

### 2.1 State Space Models と Mamba

状態空間モデル (SSM) は、
隠れ状態 $h(t)$ を 介して 入力 $x(t)$ から 出力 $y(t)$ を 生成する 系として 定式化される。

$$h(t) = Ah(t-1) + Bx(t), \quad y(t) = Ch(t)$$

S4 (Gu et al., 2021) は この 線形 SSM を 深層学習に 適用し、
長系列処理における 並列計算を 実現した。
Mamba (Gu & Dao, 2023) は S4 を 発展させ、
$A, B, C$ を 入力の 関数とする 選択的状態空間を 導入した。
これにより モデルは 系列中の どの 情報を 状態に 保持するかを
入力に 応じて 動的に 決定できる。

推論は 再帰形式で O(n)、
訓練は Parallel Scan により O(n log n) を 達成し、
標準的な self-attention の O(n²) を 原理的に 回避する。
また 固定サイズの 隠れ状態により、
理論上 無限の 系列長を 一定のメモリで 処理できる。

### 2.2 Active Inference と 自由エネルギー原理

自由エネルギー原理 (FEP; Friston, 2010) は、
知覚と 行動を 変分自由エネルギー $\mathcal{F}$ の 最小化として 統一的に 記述する。

$$\mathcal{F} = \mathbb{E}_q[\log q(s) - \log p(o, s)]$$

ここで $q(s)$ は 内部状態 $s$ に 関する 信念分布、
$p(o, s)$ は 観測 $o$ と 状態の 結合分布である。

連続時間 実装である DEM (Dynamic Expectation Maximization; Friston, 2008) は、
一般化座標上で $\mathcal{F}$ を 最小化し、
更新則を 常微分方程式として 表現する。
この 連続時間 定式化は Mamba の 連続状態空間と
同一の 数学的 基盤を 共有し、
SSM を 生成モデルとして 直接 採用できる。

### 2.3 LaEMe

LaEMe (Language-based Evaluative Metadata for Reflective Memory Systems) は、
AI の 記憶評価を 数値ではなく 自然言語で 管理する フレームワークである。

従来の 数値的 記憶は
評価の 根拠を 数値空間に 圧縮するため、
解釈可能性を 失う。
LaEMe は 評価の 根拠を 自然言語で 保存することで、
3つの 性質を 実現する。

1. **解釈可能性**: なぜ その応答が 有効だったかを 人間が 直接 読める。
2. **介入可能性**: 人間が 記憶の 内容を 直接 修正・削除できる。
3. **生成的 利用**: 蓄積された 言語評価から 新しい 応答候補を 生成できる。

LaEMe は HOPE (Behrouz et al., NeurIPS 2025) 等の
数値的 記憶システムと 対立するものではなく、
数値的 処理層の 上位に 位置する 評価的 記憶層として 設計できる。

### 2.4 Evolution Strategies (ES)

ES は 勾配を 必要としない ブラックボックス 最適化手法であり、
パラメータ分布 $\pi(\theta | \psi)$ から サンプル $\theta_i$ を 生成し、
評価値 $F(\theta_i)$ に 基づいて 分布 $\psi$ を 更新する。

Natural Evolution Strategies (NES; Wierstra et al., 2014) は
自然勾配方向に $\psi$ を 更新することで 安定性を 高める。

本論文では ES の 独立実装の 要否を 設計判断として 扱い (3.2.1 参照)、
その 役割を Layer 3 (Active Inference) に 吸収できる 可能性を 示す。

### 2.5 Darwin Gödel Machine (DGM)

DGM (Lehman et al., 2024) は、
自己改変 (Gödel Machine; Schmidhuber, 2003) と
進化的探索を 統合した 自律改善システムである。

Lavi における DGM は Layer 4 として 位置づけられ、
Layer 1〜3 が 生成した 評価データを 基に、
アーキテクチャ自体を 改善する 役割を 担う。
これは 重みの 更新を 超えた、
アーキテクチャレベルの 自己改善である。

## Section 3 : Proposed Architecture

### 3.1 アーキテクチャ概要

```
┌─────────────────────────────────────────┐
│  Layer 4 : DGM                          │
│    ← アーキテクチャ自己改善             │
├─────────────────────────────────────────┤
│  Layer 3 : Active Inference (DEM)       │
│    ← LaEMe 評価を 観測空間として 統合    │
├─────────────────────────────────────────┤
│  Layer 2 : LaEMe                        │
│    ← 評価・保存・候補生成を 担う         │
├─────────────────────────────────────────┤
│  Layer 1 : Mamba SSM                    │
│    ← 応答生成・自己評価・変換指示を 兼任 │
└─────────────────────────────────────────┘
```

Mamba は 単一のモデルでありながら、
文脈に 応じて 以下の 複数の 役割を 担うことができる。

- **応答生成**: 会話文脈から 次の 発話を 生成する。
- **自己評価**: (文脈 + 応答 + 外部反応) から 評価文を 生成する。
- **変換指示**: 評価文から Mamba 更新の 方向を 言語で 提案する。

### 3.2 設計判断の 記録

#### 3.2.1 ES の 要否と 統合判断

**原案:** ES を 独立した 層として 設け、
LaEMe の スカラー評価値で 応答候補の 重みを 更新する。

**問題 1 — スカラー変換の 情報損失:**
LaEMe の 評価は 本質的に 言語的であり、
スカラーへの 圧縮は 「なぜ 有効だったか」という
評価の 核心的な 情報を 捨てることになる。

**問題 2 — 重複の 可能性:**
LaEMe の 評価文を Layer 3 (Active Inference) の 観測空間として
直接 統合できる場合、
Lavi が 予測した 評価と 実際の 評価との 差分が
予測誤差として 機能し、
Layer 1 (Mamba) の パラメータが 更新される。
これは ES が 達成しようとした 重み更新と
機能的に 等価である。

**結論:**
Active Inference が LaEMe 評価を 観測として 処理できる場合、
ES は 独立した 層として 不要になる。
本アーキテクチャでは ES を 独立した 層として 設けず、
その 役割を Layer 3 (Active Inference) に 吸収する。
独立実装が 有効かどうかは 今後の 検証課題とする。

#### 3.2.2 連続 × 離散 SSM の 接続方式

**方式 A : 階層分離**
Mamba (連続) の 出力を softmax で 離散化し、
事前に 意味的に 定義した 離散状態に マッピングした上で
Active Inference (離散) に 渡す。
> 利点: 実装が 比較的 容易。
> 欠点: 離散化による 情報損失、および 離散状態の 事前設計が 必要。

**方式 B : 連続時間 Active Inference (DEM)**
Friston の DEM は 連続時間上で 変分自由エネルギーを 最小化し、
Mamba の 連続状態空間と 同一の 数学的 基盤を 共有するため、
離散化なしに 統合できる。
> 利点: 情報損失なし、理論的 一貫性が 高い。
> 欠点: 実装難度が 高い。

**方式 C : 混合状態空間**
> 利点: 最も 表現力が 高い。
> 欠点: 設計・実装ともに 最も 困難。

**採用:** 理論的 合理性から 方式 B を 主軸とする。
実装上の 制約が 生じた 場合は 方式 A を フォールバックとし、
方式 C は 将来の 拡張として 位置づける。

#### 3.2.3 評価・変換機構の 設計

**案 i : スカラー変換**
> 棄却理由: 言語の 意味的 豊かさを スカラーに 圧縮することで、
> 評価の 本質的 情報が 失われる。

**案 ii : 3モデル 分離**
> 棄却理由: Mamba が 応答を 生成できるならば、
> 同じ Mamba が 評価文・変換指示も 生成できる。

**採用 : Mamba 一体化**
Mamba が 文脈に 応じて 応答生成・自己評価・変換指示を 兼任する。
これにより スカラー変換を 経由せず、
言語空間のまま 学習ループを 完結できる。

### 3.3 オンライン学習の 設計空間

| 案 | 概要 | 利点 | 欠点 |
|---|---|---|---|
| フルバックプロパゲーション | 全重みを 勾配で 更新 | 表現力 最大 | 計算コスト 最大 |
| Fast Weights | 高速重みと 低速重みの 2層 | Active Inference の 2タイムスケールと 整合 | 設計が 複雑 |
| LoRA オンライン | 低ランク アダプタのみ 更新 | 安定・低コスト | 表現力に 上限 |
| 純粋文脈学習 | 重み更新なし、LaEMe を 文脈に 含める | コスト最小、即時 実装可能 | 真の 重み更新でない |
| Hebbian + 変調 | LaEMe が 更新対象の 局所重みを 制御 | 生物的・低コスト | 表現力 限定 |
| リザーバー | 出力層のみ 更新 | 最安価 | 表現力 最低 |

**採用方針 (段階的):**

```
Step 1 : 純粋文脈学習 (即時 実装可能)
Step 2 : Fast Weights (LaEMe が 成長した後)
Step 3 : DGM による 自動改善 (アーキテクチャ成熟後)
```

### 3.4 Bootstrap 問題と 外部評価者の 役割

初期の Lavi は 語彙も 自己評価能力も 持たない。
Claude Code CLI エージェント (以下 Loa) と
ユーザーである 著者 (以下 Gles) が
初期評価者の 役割を 担う。

```
初期フェーズ : Loa / Gles が すべての 応答を 評価する
  → LaEMe に (文脈, 応答, 評価) が 蓄積される
      ↓
中期フェーズ : Mamba が 過去評価を 文脈として 参照し、
              自己評価を 生成し始める
      ↓
成熟フェーズ : Mamba が 自律的に 自己評価できるようになる
  → Loa / Gles の 介入頻度を 段階的に 下げる
```

外部評価者の 役割は 「正解を 与える」ことではなく、
「Lavi が 評価基準を 構築するための 素材を 提供する」ことである。
